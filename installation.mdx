### Browser features required

The platform uses these modern web technologies:

- **ES6+ JavaScript**: For hash-based routing and dynamic content loading
- **Fetch API**: For loading pages and external data (motivational quotes)
- **CSS Grid and Flexbox**: For responsive layouts
- **HTML5 semantic elements**: For accessibility and structure
- **localStorage**: For storing user preferences (theme, font size)

```javascript
// Example: Hash-based routing implementation
async function loadView(path) {
  const res = await fetch(path);
  const html = await res.text();
  if (main) main.innerHTML = html;
}

// Route handling
window.addEventListener("hashchange", hashRoutes);
```

## System architecture

Understanding how UniJAC is structured can help you make the most of the platform:

### Single-page application

UniJAC uses a single `index.html` file with hash-based routing to create a multi-page experience:

```javascript
// Main content container
const main = document.querySelector("#mainContent");

// Hash route handler
async function hashRoutes() {
  const route = location.hash || "#/home";
  
  switch (route) {
    case "#/home":
      if (main) main.innerHTML = homeHTML;
      initHome();
      break;
    case "#/register":
      await loadView("pages/register.html");
      break;
    // ... other routes
  }
}
```

### Performance optimization

UniJAC implements several performance optimizations:

**Lazy loading images**

```javascript
const images = document.querySelectorAll('[data-src]');
const options = { rootMargin: "200px" };

const observer = new IntersectionObserver((entries, self) => {
  entries.forEach((entry) => {
    if (entry.isIntersecting) {
      entry.target.src = entry.target.getAttribute("data-src");
      self.unobserve(entry.target);
    }
  });
}, options);

images.forEach((image) => observer.observe(image));
```

**Preloaded assets**

```html
<link
  rel="preload"
  href="./css/index.css"
  as="style"
  onload="this.onload = null; this.rel = 'stylesheet';"
/>

<link
  rel="preload"
  href="/assets/fonts/InterVariable.woff2"
  as="font"
  type="font/woff2"
  crossorigin
  font-display="swap"
/>
```

## Troubleshooting

<AccordionGroup>
  <Accordion title="Page not loading">
    1. Check your internet connection
    2. Refresh the page (Ctrl+R or Cmd+R)
    3. Clear your browser cache
    4. Try a different browser
  </Accordion>
  
  <Accordion title="Images not appearing">
    Images use lazy loading. Scroll down the page to trigger image loading, or wait a moment for the Intersection Observer to activate.
  </Accordion>
  
  <Accordion title="Navigation not working">
    UniJAC uses hash-based routing. Make sure:
    - JavaScript is enabled in your browser
    - You're not blocking scripts
    - Your browser supports modern JavaScript (ES6+)
  </Accordion>
  
  <Accordion title="Modal won't close">
    If the welcome modal or other overlays won't close:
    1. Try clicking the overlay background
    2. Press the Escape key
    3. Refresh the page if needed
  </Accordion>
</AccordionGroup>

## Developer setup (optional)

If you're a developer wanting to run UniJAC locally:

<Steps>
  <Step title="Clone the repository">
    ```bash
    git clone https://github.com/alexander10pb/UniJac.git
    cd UniJac
    ```
  </Step>
  
  <Step title="Install dependencies">
    ```bash
    npm install
    ```
  </Step>
  
  <Step title="Run locally">
    Since UniJAC is a static site, you can use any local server:
    
    ```bash
    # Using Python
    python -m http.server 8000
    
    # Using Node.js http-server
    npx http-server
    
    # Using VS Code Live Server extension
    # Right-click index.html > Open with Live Server
    ```
  </Step>
</Steps>

<Note>
  Local development is only needed if you're contributing to the platform or customizing it for your organization. Regular users should access the production URL directly.
</Note>
