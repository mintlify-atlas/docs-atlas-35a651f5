## Overview

UniJAC is designed with accessibility at its core, ensuring all residents can effectively use the platform regardless of visual, motor, or cognitive abilities. The platform implements multiple accessibility features controlled through a dedicated menu.

## Font Size Controls

UniJAC provides dynamic font scaling for improved readability:

### Implementation

```javascript index.js:85-107
const btnAumentar = document.querySelector("#aumentarFuente");
const btnDisminuir = document.querySelector("#disminuirFuente");

// Tamano base de fuente
let tamanoFuente = 16;

if (btnAumentar && btnDisminuir) {
  const html = document.documentElement;
  
  btnAumentar.addEventListener("click", () => {
    if (tamanoFuente <= 25) {
      tamanoFuente += 1;
      document.documentElement.style.fontSize = `${tamanoFuente}px`;
    }
  });
  
  btnDisminuir.addEventListener("click", () => {
    if (tamanoFuente >= 14) {
      tamanoFuente -= 1;
      document.documentElement.style.fontSize = `${tamanoFuente}px`;
    }
  });
}
```

### Font Size Range

<CardGroup cols={3}>

</Tip>

### How It Works

<Steps>

  <Step title="Select Font Control">

</Steps>

## Contrast Mode

High contrast mode improves visibility for users with low vision:

### Toggle Implementation

```javascript index.js:108-114
const btnContraste = document.querySelector("#contraste");

if (btnContraste) {
  btnContraste.addEventListener('click', () => {
    btnContraste.classList.toggle('active');
    
    const current = html.getAttribute('data-theme');
    html.setAttribute('data-theme', current === '' ? 'contrast' : '');
  });
}
```

### Theme Attribute

The contrast mode uses a data attribute on the root HTML element:

```html HTML Element
<html lang="es" data-theme="">
```

**Values:**
- `data-theme=""` - Normal mode (default)
- `data-theme="contrast"` - High contrast mode

<CodeGroup>
```css Expected CSS Implementation
/* Normal mode */
:root {
  --color-text: #333333;
  --color-bg: #ffffff;
  --color-primary: #3881d5;

  --color-text: #000000;
  --color-bg: #ffffff;
  --color-primary: #0066cc;
  --border-width: 2px;
}

[data-theme="contrast"] * {
  border-color: currentColor !important;
}
```

## Keyboard Navigation

UniJAC supports full keyboard navigation:

### Navigation Patterns

<Tabs>
  <Tab title="Tab Key">
    **Forward Navigation**
    
    ```
    Tab → Next interactive element
    ```
    
    **Typical order:**
    1. Skip to content link
    2. Logo
    3. Navigation menu items
    4. Main content links
    5. Form inputs
    6. Action buttons
    7. Footer links
  </Tab>
  
  <Tab title="Shift+Tab">
    **Backward Navigation**
    
    ```
    Shift+Tab → Previous interactive element
    ```
    
    Reverses through the tab order for quick access to previous elements.
  </Tab>
  
  <Tab title="Enter/Space">
    **Activation**
    
    ```
    Enter → Activate links and buttons
    Space → Toggle buttons and checkboxes
    ```
    
    Example from code:
    ```javascript
    btnMenu.addEventListener('click', (e) => {
      menu.classList.toggle('is-open');
      overlay.classList.toggle('active');
    });
    ```
  </Tab>
  
  <Tab title="Escape">
    **Close Modals**
    
    ```
    Escape → Close overlays and dialogs
    ```
    
    Closes:
    - Accessibility menu
    - Navigation menu
    - Modal dialogs
    - Dropdown menus
  </Tab>
</Tabs>

### Focus Management

The code includes proper focus handling:

```javascript index.js:19-63
document.addEventListener('click', (e) => {
  // Close menu when clicking outside
  if (!menu.contains(e.target) && !btnMenu.contains(e.target)) {
    menu.classList.remove('is-open');
    overlay.classList.remove('active');
  }
  
  // Open accessibility menu
  if (btnAccesibility && accessContent && accessContent.contains(e.target)) {
    btnAccesibility.classList.add('visible');
    overlay.classList.add('active');
  } else if (!menu.classList.contains('is-open')) {
    btnAccesibility.classList.remove('visible');
    overlay.classList.remove('active');
  }
});
```

<Info>
Clicking outside menus automatically closes them, maintaining focus context and preventing confusion.
</Info>

## Responsive Header

Header visibility adapts to scroll behavior:

```javascript index.js:67-77
let scrollPrev = window.scrollY;

window.addEventListener('scroll', () => {
  const scrollCurrent = window.scrollY;
  if (scrollCurrent > 700 && scrollCurrent > scrollPrev) {
    header?.classList.add('hide');
  } else {
    header?.classList.remove('hide');
  }
  scrollPrev = scrollCurrent;
  btnScroll?.classList.toggle('is-visible', window.scrollY >= 600);
  header?.classList.toggle('on-scroll', window.scrollY >= 50);
});
```

**Features:**
- Hides header when scrolling down beyond 700px
- Shows header when scrolling up
- Adds visual state at 50px scroll
- Improves screen real estate on mobile

## Best Practices

<AccordionGroup>
  <Accordion title="For Users with Visual Impairments">
    1. **Enable High Contrast**
       - Click accessibility icon
       - Select "Contraste" button
       - Verify all text is readable
    
    2. **Adjust Font Size**
       - Start at default 16px
       - Increase incrementally using "Aumentar fuente"
       - Find optimal size between 14px-25px
    
    3. **Use Screen Reader**
       - All content has proper ARIA labels
       - Semantic HTML provides context
       - Live regions announce dynamic content
  </Accordion>
  
  <Accordion title="For Users with Motor Impairments">
    1. **Keyboard Navigation**
       - Use Tab to navigate forward
       - Use Shift+Tab to go backward
       - Press Enter to activate links/buttons
       - Press Escape to close menus
    
    2. **Target Sizes**
       - All buttons meet minimum 44×44px target size
       - Adequate spacing between interactive elements
       - Large clickable areas for links
    
    3. **Scroll to Top**
       - Use floating button to return to top
       - Avoid excessive scrolling
       - Keyboard accessible
  </Accordion>
  
  <Accordion title="For Users with Cognitive Disabilities">
    1. **Clear Structure**
       - Consistent navigation across pages
       - Predictable button locations
       - Breadcrumb navigation shows location
    
    2. **Simple Language**
       - Clear labels and instructions
       - Icons supplement text
       - Consistent terminology
    
    3. **Focus Management**
       - Only one menu open at a time
       - Clear focus indicators
       - Logical tab order
  </Accordion>
</AccordionGroup>

## Testing Accessibility

<Steps>
  <Step title="Keyboard Only">
    Navigate the entire site using only keyboard
    
    - Can you reach all interactive elements?
    - Is the focus indicator always visible?
    - Does tab order make logical sense?
  </Step>
  
  <Step title="Screen Reader">
    Test with NVDA, JAWS, or VoiceOver
    
    - Are all images described?
    - Do form fields have labels?
    - Are page regions properly identified?
  </Step>
  
  <Step title="Font Scaling">
    Test at minimum, default, and maximum sizes
    
    - Does layout remain usable at 25px?
    - Is text still readable at 14px?
    - Do elements overlap or break?
  </Step>
  
  <Step title="Contrast Mode">
    Enable high contrast and verify:
    
    - All text has sufficient contrast (4.5:1 minimum)
    - Borders and icons are visible
    - Interactive elements are distinguishable
  </Step>
</Steps>
